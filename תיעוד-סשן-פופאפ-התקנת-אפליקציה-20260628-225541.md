# תיעוד סשן - פופאפ התקנת אפליקציה

תאריך: 2026-06-28  
נושא: ניהול סגירת פופאפ התקנת PrimeOS לפי משתמש

## רקע

המשתמש ביקש שפופאפ התקנת האפליקציה לא ימשיך להטריד אחרי לחיצה על X:

- לחיצה ראשונה תסתיר ל-24 שעות.
- לחיצה שנייה תסתיר לשבוע.
- לחיצה שלישית תסתיר לתמיד.
- הרישום ינוהל בטבלה חדשה: מי לחץ, מתי הפופאפ עלה ומתי נסגר.
- ההסבר איך להתקין שוב ידנית יופיע במסך הגדרות.
- הפתרון יעבוד גם למנהל וגם למשתמש רגיל.
- לעדכן Supabase אם צריך.

## קבצים ששונו

### `src/components/pwa/usePwaInstall.ts`

הוחלף מנגנון הסגירה הקודם, שהיה מבוסס רק על `localStorage`, במנגנון משולב:

- קריאת המשתמש המחובר דרך Supabase Auth.
- טעינת מצב הסגירה מטבלת `pwa_install_prompt_dismissals`.
- חישוב הסתרה לפי `dismiss_count`:
  - `1` -> הסתרה ל-24 שעות.
  - `2` -> הסתרה ל-7 ימים.
  - `3+` -> הסתרה קבועה.
- רישום `last_shown_at` כאשר הפופאפ מוצג בפועל.
- רישום `last_dismissed_at` כאשר המשתמש סוגר את הפופאפ.
- תמיכה ב-`permanently_dismissed`.
- שמירת fallback ב-`localStorage`, כדי שמשתמשים שכבר סגרו בעבר לא יקבלו את הפופאפ מחדש רק בגלל המעבר לטבלה.
- בעת התקנה מוצלחת, הפופאפ מסומן כמוסתר קבוע.

### `supabase/migrations/20260628200000_pwa_install_prompt_dismissals.sql`

נוספה מיגרציה חדשה שיוצרת את הטבלה:

`public.pwa_install_prompt_dismissals`

עמודות עיקריות:

- `user_id` - המשתמש שלחץ או ראה את הפופאפ.
- `prompt_key` - מזהה הפופאפ, כרגע `pwa_install`.
- `dismiss_count` - מספר סגירות.
- `dismissed_until` - עד מתי להסתיר זמנית.
- `permanently_dismissed` - האם להסתיר לתמיד.
- `last_shown_at` - מתי הפופאפ הוצג לאחרונה.
- `last_dismissed_at` - מתי נסגר לאחרונה.
- `created_at`, `updated_at`.

נוספו גם:

- primary key על `(user_id, prompt_key)`.
- index לפי `last_shown_at`.
- trigger לעדכון `updated_at`.
- RLS.
- הרשאות `select`, `insert`, `update` ל-`authenticated`.

מדיניות RLS:

- משתמש מחובר יכול לקרוא, ליצור ולעדכן רק את הרשומה של עצמו.
- מנהל יכול לקרוא את כל הרשומות.

### `src/pages/admin/UserSettingsPage.tsx`

נוסף כרטיס במסך הגדרות משתמש עם הסבר התקנה ידנית:

- ב-iPhone: לפתוח ב-Safari, ללחוץ שיתוף, לבחור "הוסף למסך הבית".
- באנדרואיד או בדסקטופ: לפתוח את תפריט הדפדפן ולבחור התקנה או הוספה למסך הבית.
- כולל הסבר על מדיניות ההסתרה: 24 שעות, שבוע, ואז לתמיד.

### `src/pages/admin/SettingsPage.tsx`

נוסף הסבר זהה/מקוצר גם במסך הגדרות מנהל, תחת אזור ההרשמה והאונבורדינג.

### `src/types/db.ts`

נוסף טיפוס TypeScript:

`PwaInstallPromptDismissal`

הטיפוס משקף את מבנה הטבלה החדשה.

## פעולות Supabase שבוצעו

בוצע dry run:

```bash
supabase db push --dry-run --include-all --password "$SUPABASE_DB_PASSWORD"
```

התוצאה הראתה שרק המיגרציה החדשה תידחף:

```text
20260628200000_pwa_install_prompt_dismissals.sql
```

לאחר מכן בוצע push בפועל:

```bash
supabase db push --include-all --password "$SUPABASE_DB_PASSWORD" --yes
```

המיגרציה הוחלה בהצלחה על ה-remote.

בוצעה בדיקת migration list:

```bash
supabase migration list --password "$SUPABASE_DB_PASSWORD"
```

והמיגרציה מופיעה גם ב-Local וגם ב-Remote:

```text
20260628200000 | 20260628200000 | 2026-06-28 20:00:00
```

ניסיון אימות ישיר דרך `supabase db query --linked` לא בוצע כי חסר `SUPABASE_ACCESS_TOKEN`. ניסיון `psql` השתמש בהגדרות localhost מה-env המקומי ולכן לא התאים לאימות ה-remote. אימות ההחלה התבסס על `supabase db push` ו-`supabase migration list`.
