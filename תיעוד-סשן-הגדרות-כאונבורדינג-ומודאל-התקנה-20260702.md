# תיעוד סשן - הגדרות כאונבורדינג ומודאל התקנה

תאריך: 2026-07-02

## מטרת השינוי

להפוך את מסך ההגדרות למסך האונבורדינג עצמו, ולהעביר את הסבר התקנת האפליקציה מכרטיס פתוח לכפתור שפותח מודאל בעברית, מיושר RTL.

## מה בוצע

1. `src/pages/OnboardingPage.tsx`
   - נוסף מצב `embedded` לרכיב `OnboardingPage`.
   - במצב מוטמע הרכיב מציג את תוכן האונבורדינג בלבד, בלי תפריט צד, תפריט תחתון ובלי `InstallPrompt` גלובלי נוסף.
   - בסיום אונבורדינג מתוך ההגדרות המשתמש נשאר במסלול `/admin/user-settings`.
   - מצב ברירת המחדל של `/onboarding` נשאר עצמאי כמו קודם.

2. `src/pages/admin/UserSettingsPage.tsx`
   - מסך ההגדרות מציג עכשיו את האונבורדינג ישירות באמצעות `<OnboardingPage embedded />`.
   - הוסר כרטיס "מעבר לאונבורדינג".
   - הוסר כרטיס הוראות התקנה פתוח.
   - נוסף כפתור "התקנת האפליקציה" בראש המסך.
   - לחיצה על הכפתור פותחת מודאל RTL עם:
     - `dir="rtl"`.
     - יישור טקסט לימין.
     - הוראות נפרדות לאייפון ול־Android/desktop.
     - ללא כפתור התקנה בתוך המודאל; ההתקנה מתבצעת דרך תפריט הדפדפן לפי ההוראות.
     - סגירה באמצעות כפתור, לחיצה על הרקע או מקש Escape.

## בדיקות שבוצעו

1. בדיקת TypeScript:

```bash
npm run typecheck
```

תוצאה: עבר בהצלחה.

2. build מלא:

```bash
npm run build
```

תוצאה: עבר בהצלחה. התקבלה אזהרת Vite קיימת על chunk גדול, ללא כשל build.

3. בדיקת טעינת מסלול בדפדפן:

```text
http://127.0.0.1:5173/admin/user-settings
```

תוצאה: המסלול נטען, אך בדפדפן הבדיקה לא היה סשן משתמש תקף ולכן האפליקציה ניתבה אל `/login`.

## מגבלות בדיקה

בדיקת UI מלאה של מסך ההגדרות לאחר התחברות לא בוצעה כי בדפדפן הבדיקה לא היה סשן Supabase תקף. בקונסול הופיעה שגיאת refresh token מהסשן המקומי הקיים:

```text
Supabase auth refresh token returned 400
```

זו מגבלת אימות בסביבת הבדיקה המקומית, לא כשל TypeScript או build.
