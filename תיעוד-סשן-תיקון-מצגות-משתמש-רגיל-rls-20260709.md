## מה נמצא — שני באגים נפרדים

### 1. RLS חסר על `deck_ai_images` (הגורם לפער אדמין/משתמש רגיל)

השקפים שנוצרו ע"י GPT Images לכל מצגת נשמרים בטבלה `public.deck_ai_images` (מטא-דאטה) ובקובץ תחת `deck-ai/<requestId>/...` בבאקט `outputs`. עד היום, ל־`deck_ai_images` היו רק שתי מדיניות RLS:

```sql
create policy "deck_ai_images_admin_select" ... using (public.is_admin());
create policy "deck_ai_images_admin_delete" ... using (public.is_admin());
```

כלומר **אף משתמש שאינו אדמין לא יכול היה לקרוא שורה אחת מהטבלה הזו** — לא משנה אם הוא יצר את הבקשה בעצמו. ה-UI (`fetchPersistedDeckImages` ב-[deck.ts](src/lib/deck.ts:323), נקרא מ-[GptImagesDeck.tsx](src/components/GptImagesDeck.tsx:109)) קיבל 0 שורות בחזרה ולכן מעולם לא פתח את עורך השקפים אוטומטית.

בנוסף, גם אם היה SELECT על הטבלה, הקובץ עצמו ב-storage לא היה נגיש: הפונקציה `user_can_read_brand_output` (מ-[20260624150000_brand_ai_images_access.sql](supabase/migrations/20260624150000_brand_ai_images_access.sql)) בדקה גישה רק דרך **חברות במותג** (`user_brands`), לא דרך יוצר הבקשה (`requests.created_by`) — בניגוד למדיניות המקבילה על `outputs` הרגילים שכן נתנה גישה ליוצר.

אימתתי את זה ישירות מול ה-DB (ראו "איך אימתנו" למטה) לפני ואחרי התיקון.

### 2. תוצרי מצגת ישנים עם JSON טכני של הבוט במקום תוכן שקפים

בדקתי את `outputs.text_content` של הבקשה שדווחה ומצאתי שהוא מכיל:

```
```json
{
  "action": "ready_to_generate",
  "message_to_user": "מעולה, יש לי את הפרטים הדרושים...",
  "brief": { "output_type": "presentation_kit", ... }
}
```
```

זה קרה כי ה-systemPrompt של בוט הוואטסאפ (שמאלץ תשובות JSON בפורמט שיחה) הוזרק גם לפרומפט של יצירת ה-**outline** של המצגת עצמה. זה **כבר תוקן בקומיט הקודם של המשתמש** (`58287d6`, 2026-07-08 18:35) ב-[worker.ts](supabase/functions/_shared/worker.ts:867) — אבל וידאתי שהפריסה לענן (`supabase functions deploy`) בפועל קדמה לקומיט בכ-13 דקות, ולכן היה סיכוי שהגרסה הפרוסה עדיין ישנה. פרסתי מחדש (ראו "מה תוקן" למטה) כדי לוודא שהקוד הרץ תואם את הקומיט.

**חשוב:** יש **14 בקשות מצגת ישנות** ב-DB עם ה-JSON הטכני הזה ב-`text_content` (השאילתה ששימשה לספירה נמצאת ב"איך אימתנו"). לא תוקנו רטרואקטיבית — כשמשתמש נכנס לעריכה של אחת מהן, `openContentEditor`/`applyContentEdits` (ב-[GptImagesDeck.tsx](src/components/GptImagesDeck.tsx:287)) כבר בונה מחדש מהבריף ושומר Markdown תקין, כך שהתוכן "מתרפא" ברגע שנפתח לעריכה. אם צריך תיקון גורף מיידי לכל ה-14 — זו עבודה נפרדת (סקריפט מיגרציה חד-פעמי שקורא ל-`generate-presentation` מחדש לכל אחת).
