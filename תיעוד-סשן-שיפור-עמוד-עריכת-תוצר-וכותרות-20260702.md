# תיעוד סשן: שיפור עמוד עריכת תוצר וכותרות מערכת

תאריך: 2026-07-02

## מטרת הסשן

להוריד עומס קוגניטיבי בעמוד עריכת תוצר (`/admin/files/:requestId/revise`) וליישר את כותרות העמודים במערכת לסגנון שקט, תפעולי ופחות "AI-generated".

## עמוד עריכת תוצר

קובץ מרכזי: `src/pages/admin/RevisePage.tsx`

### מבנה הצד השמאלי

הצד השמאלי עבר ממבנה שטוח של כמה טפסים שווי משקל למבנה היררכי:

- אזור `הפצה ושיתוף` מוצג ראשון.
- אזור `עריכה ושיפור` מוצג מתחתיו כאקורדיון.
- פעולת "רוצים ... אחר לגמרי?" עברה לתוך אזור העריכה, מיד מתחת לכפתור העדכון.

### הפצה ושיתוף

הכפתורים מוצגים מיד, בלי אקורדיון נוסף:

- אינסטגרם
- פייסבוק
- מייל

הסדר עודכן לפי בקשת המשתמש: אינסטגרם → פייסבוק → מייל.

הכפתורים קיבלו צבעים מובחנים:

- אינסטגרם: ורוד
- פייסבוק: כחול
- מייל: ירוק

לחיצה על מייל פותחת את שדה כתובת המייל וכפתור השליחה מתחת לשורת הערוצים.

### אייקונים

נבדקה המלצת UX לגבי אייקונים:

- להשאיר טקסט גלוי לצד אייקון.
- לא להשתמש באייקונים דקורטיביים גדולים.
- אייקונים בכפתורי פעולה כן עוזרים לסריקה.

בעקבות זאת נוסף אייקון שיתוף קטן ועדין ליד כותרת `הפצה ושיתוף`, בצבע muted, בלי להפוך אותו לאלמנט דומיננטי.

## SocialScheduleSection

קובץ: `src/components/SocialScheduleSection.tsx`

בוצעו שינויים כדי לתמוך בפריסת הערוצים:

- כפתורי פייסבוק/אינסטגרם הפכו מצמד כפתורים רחבים לצ'יפים קומפקטיים.
- נוספה אפשרות להעביר `trailingAction`, כדי שכפתור המייל יופיע באותה שורה אחרי פייסבוק.
- נוספה אפשרות להעביר `title=""`, כך שכותרת פנימית לא תופיע כשהרכיב משולב בתוך אזור `הפצה ושיתוף`.
- לחיצה חוזרת על אותו ערוץ סוגרת את אזור התזמון שלו.

## כותרת עמוד עריכת תוצר

הכותרת `שיפור תוצר` הוחלפה בכותרת ספציפית לפי סוג התוצר:

- `עריכת טקסט`
- `עריכת תמונה`
- `עריכת מסמך`
- `עריכת מצגת`

הסגנון הוחלף מ־`text-2xl font-bold` לסגנון שקט יותר:

```tsx
text-xl font-semibold tracking-normal
```

נוסף משפט משנה:

```text
עדכנו את התוצר, שתפו או שמרו גרסה חדשה.
```

## כותרות בשאר העמודים

בוצע מעבר על עמודי `src/pages` ו־`src/pages/admin`, והכותרות המרכזיות הותאמו לסגנון page header תפעולי:

```tsx
<h1 className="text-xl font-semibold tracking-normal">...</h1>
```

במסכי עבודה פנימיים הורדו כותרות מ־`text-2xl font-bold`, `text-[28px] font-bold` ו־`font-extrabold` לסגנון שקט יותר.

## עקרונות UX שיושמו

- כותרת עמוד צריכה להגיד איפה המשתמש נמצא, לא למכור פיצ'ר.
- במסכי עבודה פנימיים יש להעדיף כותרות שקטות על פני כותרות hero.
- כפתור ראשי אחד צריך להישאר דומיננטי; פעולות הפצה הן משניות.
- אייקונים צריכים להופיע לצד טקסט, לא במקום טקסט.
- לא להשתמש באייקונים דקורטיביים גדולים לכותרות משניות.
- להפחית קווי הפרדה וכפתורים רחבים כשאין להם משקל פעולה ראשי.

## בדיקות

הורצה בדיקת TypeScript:

```bash
./node_modules/.bin/tsc --noEmit
```

הבדיקה עברה תקין.
