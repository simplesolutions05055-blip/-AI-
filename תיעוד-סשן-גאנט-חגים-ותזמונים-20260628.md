# תיעוד סשן - גאנט חגים ותזמונים באדמין

תאריך: 2026-06-28

## מטרת הסשן

להוסיף לאדמין מסך גאנט חודשי שמציג חגי ישראל לפי תאריך, מאפשר לפתוח כל יום, לראות אירועים באותו יום, ולהוסיף תזמון פרסום לפייסבוק או אינסטגרם לפי מותג.

## מקור נתונים לחגים

נבחר Hebcal API:

- חינמי
- לא דורש API key
- תומך בלוח ישראלי דרך `i=on`
- מחזיר JSON עם תאריך, שם חג בעברית/אנגלית, קטגוריה ונתוני מקור

דוגמה:

```text
https://www.hebcal.com/hebcal?v=1&cfg=json&year=2026&maj=on&min=on&mod=on&i=on
```

## שינויי Supabase - חגים

נוצרה מיגרציה:

```text
supabase/migrations/20260628192513_israel_holidays_calendar.sql
```

נוצרה טבלה:

```text
public.israel_holidays
```

שדות מרכזיים:

- `date` - תאריך החג
- `title` - שם באנגלית
- `hebrew_title` - שם בעברית
- `category` / `subcategory` - סיווג מ-Hebcal
- `is_major` - סימון חג מרכזי
- `raw` - JSON מקורי מ-Hebcal
- `source` + `external_id` - מפתח upsert יציב

הופעל RLS:

- אדמין מחובר יכול לקרוא ולנהל.
- service role יכול לייבא ולעדכן.
- משתמש רגיל לא מקבל גישה דרך UI.

המיגרציה נדחפה ל-remote DB:

```bash
supabase db push --yes
```

## יבוא נתונים

נוסף סקריפט:

```text
scripts/import-israel-holidays.mjs
```

הסקריפט:

- מושך חגים מ-Hebcal
- מסנן `category === "holiday"`
- עושה upsert ל-Supabase
- משתמש ב-`SUPABASE_URL` וב-`SUPABASE_SERVICE_ROLE_KEY`

הורץ יבוא לשנים:

```text
2026, 2027, 2028
```

תוצאה:

```text
Imported 176 Israel holiday rows for 2026, 2027, 2028
```

בוצעה בדיקת REST וקיבלנו נתונים תקינים כמו פורים, ערב פסח וט״ו בשבט.

## שינויי Supabase - תזמונים לפי מותג

נוצרה מיגרציה:

```text
supabase/migrations/20260628194214_scheduled_social_posts_brand_id.sql
```

נוסף שדה:

```text
scheduled_social_posts.brand_id
```

השדה מקושר ל:

```text
public.brands(id)
```

נוסף אינדקס:

```text
scheduled_social_posts_brand_idx
```

מטרת השינוי: תזמון שנוצר מהגאנט יכול להיות משויך למותג גם בלי `request_id` או `output_id`.

המיגרציה נדחפה ל-remote DB:

```bash
supabase db push --yes
```

## Edge Function

עודכנה הפונקציה:

```text
supabase/functions/schedule-social-post/index.ts
```

השינוי:

- מקבלת `brand_id`
- שומרת אותו ב-`scheduled_social_posts`
- ממשיכה להשתמש באותו מנגנון תזמון קיים לפייסבוק/אינסטגרם

הפונקציה נפרסה:

```bash
supabase functions deploy schedule-social-post --use-api
```

## התנהגות מסך הגאנט

המסך מציג:

- חודש נוכחי בלבד בטעינה ראשונה
- מעבר חודש אחורה/קדימה
- שבוע ישראלי: ראשון עד שבת
- חגים בתוך ימי החודש
- כל יום בלוח לחיץ

הוסרו:

- KPI על מספר חגים
- תצוגת 12 חודשים שנתית
- כותרת “חגי ישראל”

## Modal אירועי יום

לחיצה על יום פותחת Modal RTL.

המודל מציג:

- כותרת לפי יום וחודש
- רשימת חגים/אירועים באותו יום
- שם החג בעברית
- שם באנגלית קטן אם קיים
- כפתור `הוספת תזמון`

הוסרו מהמודל:

- הסברי חג
- קישור “מידע נוסף”
- תגיות כמו `חג`, `modern`, `major`

אם אין אירועים ביום:

- מוצג מצב ריק: אין אירועים או חגים שמורים ליום הזה.

## הוספת תזמון מהגאנט

נוסף כפתור:

```text
הוספת תזמון
```

הלחיצה פותחת Modal חדש:

1. בחירת מותג פעיל
2. בחירת ערוץ:
   - פייסבוק
   - אינסטגרם
3. פתיחת אותו תהליך תזמון רגיל

ברירת מחדל לתאריך:

- 09:00 ביום שנבחר
- אם הזמן כבר עבר, שעה קדימה מהזמן הנוכחי

## התאמות מובייל

בוצעו התאמות למודלים:

- Modal אירועי יום כ-bottom sheet במובייל
- Modal הוספת תזמון מותאם לרוחב מסך קטן
- Modal תזמון פייסבוק/אינסטגרם מותאם למובייל
- Modal בחירת תוצרים מותאם למובייל
- תוכן פנימי לא רחב מדי
- `width: calc(100vw - 1.5rem)` למניעת גלישה
- פחות padding במובייל
- שדות וכפתורים ברוחב מתאים
- גלילה פנימית בתוך המודל
- כפתורי פעולה קבועים בתחתית
