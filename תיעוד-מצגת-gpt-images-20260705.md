# תיעוד: מצגת עם GPT Images — 2026-07-05

> **עדכון 2026-07-06:** נוסף מנגנון **הפקה ושליחה במייל ברקע** (edge function `deck-email`). ראו בתחתית המסמך.

## מה נבנה

מקטע חדש בשם **"מצגת עם GPT Images"** בעמוד התוצר של מצגת — מופיע **גם בהפקה חדשה וגם במסך עריכה (revise)**, כי שניהם מרנדרים את אותו רכיב (`DeckExport`).

הזרימה המלאה (E2E):

1. המשתמש כותב **אילו שקפים להפיק** — למשל `1-5` או `1,3,7` — כדי לבחון את התוצאה לפני הפקת כל המצגת.
2. המערכת טוענת את תוכן השקפים מהבריף (אותו מנוע קיים).
3. לכל שקף בטווח נוצרת **תמונת GPT ייעודית** לפי הבריף + פלטת המותג + הלוגו האמיתי כרפרנס.
4. נפתח **מודל אישור** (כמו ביצירת תמונה לפוסט): אישור / יצירה מחדש עם הערה חופשית / השארת שקף בלי תמונה.
5. בניית הקובץ: **PPTX** (פתוח לעריכה) או **PDF** — עם פריסות עשירות.
6. **פאנל עלות** מציג כמה עלתה כל ההפקה של הבקשה (תמונות + טקסט + סה"כ).

## פריסות העיצוב (סט סגור, נקבע בקוד — לא ע"י AI)

| שקף | פריסה |
|---|---|
| שקף 1 (שער) עם תמונה | תמונת GPT כרקע מלא (full-bleed) + שכבת כהות + כותרת לבנה מיושרת ימינה |
| שקף תוכן עם תמונה | חצי-חצי: תמונה בצד שמאל על כל הגובה + פס מותג מפריד, טקסט בצד ימין |
| שקף בלי תמונה | הפריסה הקלאסית הקיימת (כותרת + בולטים) |

## RTL — מה נאכף

- `rtl="1"` + `algn="r"` ברמת **כל פסקה ופסקה** ב-PPTX (לא רק ברמת התיבה — לקח מתוקן מהעבר).
- בולטים מרובעים בצד ימין (`buChar` קוד 25AA), אינדנטציה ימנית.
- כל ה-UI החדש (סקשן, מודל, פאנל עלות) `dir="rtl"`.
- ב-PDF הרינדור הוא HTML בדפדפן → עיצוב עברית מושלם מובנה.

## חיסכון בעלויות מובנה

- **תמונות נשמרות** (`deck_ai_images` + bucket `outputs`): הרצה חוזרת של אותו טווח לא יוצרת ולא מחייבת שוב — מופיע תג "קיימת — ללא עלות".
- זרימת עבודה מומלצת: קודם `1-2` לבדיקת כיוון → אחר-כך שאר השקפים. התמונות הראשונות ממוחזרות אוטומטית.
- "יצירה מחדש" (💰) היא היחידה שמחייבת שוב — תמונה אחת בכל פעם.
- הערה: **תוכן הטקסט** של השקפים נטען מחדש בכל כניסה לעמוד (קריאת טקסט זולה, ~$0.02) — התמונות לא.

## תיעוד עלות

פאנל "עלות הפקת המצגת" קורא מ-`usage_events` לפי `request_id`:
- תמונות GPT (כמות + מספר קריאות)
- טקסט — בריף, תוכן שקפים וכתוביות
- סה"כ בדולרים

**תיקון חישוב:** עד היום כל תמונה נרשמה כ-$0.04 קבוע. עכשיו הרישום לפי הגודל והאיכות בפועל (`imageUnitCost`): למשל medium 1024×1024 = $0.042, high 1536×1024 = $0.25.

> הרשאות: `usage_events` נקרא לפי RLS של אדמין — למשתמש רגיל הפאנל פשוט לא יוצג.

# עדכון 2026-07-06: הפקה ושליחה במייל ברקע

## מה נוסף
בחלק של "מצגת עם GPT Images", אחרי אישור התמונות, נוסף אזור **"הפקה ושליחה במייל (ברקע)"**:
- תיבת מיילים — כברירת מחדל מוקלד **המייל של המשתמש המחובר** (מ-`useProfile`).
- כפתור **"+ הוספת מייל"** — אפשר להוסיף כמה נמענים שרוצים, וגם להסיר.
- כפתור הפקה → מפעיל edge function שרץ **ברקע בשרת**. אפשר **לסגור את הדפדפן** — ההפקה ממשיכה ב-Supabase, וכשמסתיימת נשלח מייל עם הקבצים לכל הנמענים.
- חיווי חי (לא מודל) במסך כל עוד רץ; בסיום "✓ נשלח בהצלחה".

## איך זה עובד (ארכיטקטורה)
`supabase/functions/deck-email/index.ts` (edge, רקע):
- `action: 'start'` → ולידציה, יצירת job (`deck-email-jobs/{jobId}.json` בבאקט outputs), `EdgeRuntime.waitUntil(runJob)`, מחזיר `{ jobId }` מיד. **הבנייה עצמאית מהדפדפן.**
- `runJob`:
  1. טוען מותג (שם, פלטה, לוגו כ-dataURL).
  2. טוען את התמונות המאושרות מ-`deck_ai_images` לפי slide_index (הכי חדש לכל שקף) — **לא מייצר תמונות מחדש, לא מחייב שוב**.
  3. בונה **PPTX בשרת** עם `npm:pptxgenjs` (עובד ב-Deno, ללא DOM) — אותן פריסות RTL (שער full-bleed, split).
  4. בונה **PDF** מ-HTML דרך `renderPdfBase64` (Vercel chromium, אותו נתיב כמו PDF של וואטסאפ).
  5. מעלה את הקבצים ל-`deck-email-jobs/{jobId}/deck.pptx|pdf` (גיבוי/אימות).
  6. שולח מייל לכל נמען בנפרד (Resend, `sendDeliverableEmail`) עם שני הקבצים מצורפים.
- `action: 'status'` → קורא את ה-job → `{ status, sentTo, pdf }` (לחיווי בלבד).

לקוח: `src/components/GptImagesDeck.tsx` — תיבת מיילים + `deck-email` start/status polling (כל 4 שניות).

## מגבלה פתוחה — PDF (בעיית תשתית, לא קוד)
שלב ה-PDF מחזיר `render-pdf 401: unauthorized` — ה-`INTERNAL_API_SECRET` ב-**Vercel** לא תואם לזה ש-Supabase שולח (אותה בעיה מ-`pdf-docx-render-secret`). לכן כרגע:
- **PPTX נשלח תמיד** (נבנה מקומית ב-Supabase, לא תלוי ב-Vercel).
- **PDF נכשל בשקט** — הפונקציה לא נכשלת, שולחת PPTX בלבד, והלקוח מציג "נשלח PPTX בלבד — הפקת ה-PDF נכשלה".
- **תיקון:** להגדיר ב-Vercel env את `INTERNAL_API_SECRET` **בדיוק** כמו הערך ב-Supabase secrets. ברגע שיתואם — ה-PDF יצורף אוטומטית, בלי שינוי קוד. (זו אותה הגדרה שנדרשת גם ל-PDF/DOCX של וואטסאפ.)
