## חלק 1 — מה נבנה היום (עובד כבר עכשיו, בלי מספר)

כל "המוח" של בוט־בקבוצה קיים, נפרס לפרודקשן ונבדק. מה שחסר הוא רק "הצינור" —
חיבור מספר WhatsApp אמיתי (חלק 2 במסמך זה).

### 1. מילת הפעלה (טריגר)

בקבוצה הבוט **לא** מגיב לכל הודעה — רק להודעות שמתחילות במילת ההפעלה:

| הודעה בקבוצה | מה הבוט עושה |
|---|---|
| `גרפיקה תכין לי פוסט למבצע קיץ` | עונה — מריץ את הבקשה |
| `@גרפיקה, מצגת על החברה` | עונה — גם עם @ ופסיק |
| `גרפיקה` (לבד) | עונה — פותח את התפריט הראשי |
| `מי בא לצהריים?` | **שותק לגמרי** |

מילת ההפעלה ניתנת לשינוי במסך ההגדרות (ברירת מחדל: `גרפיקה`).

### 2. שיחה משותפת אחת לכל הקבוצה

הקבוצה כולה מנהלת **סשן אחד משותף מול הבוט** — זה כל הקטע של קבוצה: משתמש X
מזמן את הבוט ("גרפיקה"), משתמש Y יכול לענות לתפריט ("1") והזרימה ממשיכה.
כשהבוט "עסוק" (תפריט פתוח / ברייף באמצע) כל הודעה בקבוצה נכנסת לסשן גם בלי
מילת ההפעלה; כשהוא רדום — רק מילת ההפעלה מעירה אותו.

### 3. אותו מוח בדיוק

הודעת קבוצה שעברה את הטריגר נכנסת **לאותו מנוע קיים** (תפריטים, ברייף, הפקה,
תיקונים) — לא נבנתה לוגיקה חדשה. התוצר חוזר **לתוך הקבוצה** (טקסט ותמונות).

### 4. מצב קבוצה בסימולטור 👥

בעמוד *סימולטור שיחה* נוסף מתג **"מצב קבוצה"**. כשהוא דולק, הסימולטור מתנהג
כמו קבוצה אמיתית:
- הודעה בלי מילת ההפעלה → הבוט מסביר שהוא שותק בכוונה.
- הודעה עם מילת ההפעלה → הזרימה המלאה הרגילה.

**כך בודקים את כל החוויה עוד היום, בלי אף מספר.**

### 5. הגדרות אדמין

*הגדרות מערכת ← WhatsApp ← "בוט בקבוצות"*:
- מתג **הפעלת הבוט בקבוצות** (ברירת מחדל: כבוי — הצינור האמיתי לא פעיל עד שמדליקים).
- שדה **מילת הפעלה**.

### 6. הצינור מוכן מראש

נבנתה ונפרסה פונקציה `whatsapp-group-webhook` שמחכה לשער הקבוצות (Whapi).
היא כבר יודעת: לאמת סוד משותף, לסנן רק הודעות קבוצה עם טריגר, להריץ את המנוע,
ולהחזיר תשובות ותמונות לקבוצה. ברגע שמחברים מספר — הכול עובד.

### קבצים טכניים (למפתח)

| קובץ | תפקיד |
|---|---|
| `supabase/functions/_shared/group.ts` | הליבה: טריגר, שיחות קבוצה, שליחה ל-Whapi |
| `supabase/functions/whatsapp-group-webhook/index.ts` | נקודת הכניסה מהשער (Whapi) |
| `supabase/functions/_shared/worker.ts` + `flow.ts` | ניתוב תשובות/מדיה לקבוצה |
| `supabase/functions/simulator-message/index.ts` | מצב קבוצה בסימולטור |
| `supabase/migrations/20260710150000_whatsapp_groups.sql` | עמודות group_id/group_sender + הגדרה |
| `src/pages/admin/SettingsPage.tsx` / `SimulatorPage.tsx` | ממשקי האדמין |

### בדיקות שבוצעו (על הפרודקשן)

- ✅ הודעת קבוצה בלי טריגר → `ignored: true`, אפס תגובות.
- ✅ `גרפיקה היי` → נפתחה שיחת קבוצה עם `group_id`/`group_sender`, התפריט הראשי חזר.
- ✅ ה-webhook החדש דוחה קריאות בלי הסוד המשותף (HTTP 403).
- ✅ `tsc --noEmit` נקי; המיגרציה הוחלה; כל הפונקציות המושפעות נפרסו.

## חלק 2 — המדריך: מה עושים כשמחברים מספר (E2E)

זה התהליך המלא מהרגע שמחליטים להפעיל קבוצה אמיתית. מבוסס על **דרך C**
(שער לא־רשמי — הדרך היחידה לקבוצה קיימת; ראו סיכונים בתוכנית המקורית).

### שלב 0 — מה צריך להשיג

1. **מספר טלפון חדש וייעודי** (SIM או מספר וירטואלי). ⚠️ לא המספר העסקי ולא האישי —
   מספר לא־רשמי עלול להיחסם על ידי WhatsApp.
2. **חשבון Whapi.cloud** (יש תוכנית ניסיון; בהמשך מנוי חודשי של עשרות דולרים).

### שלב 1 — חיבור המספר ל-Whapi (~חצי שעה)

1. מתקינים WhatsApp רגיל על מכשיר עם המספר החדש.
2. נרשמים ל-[Whapi.cloud](https://whapi.cloud) ויוצרים **Channel** חדש.
3. Whapi מציג קוד QR — סורקים אותו מהמכשיר:
   *WhatsApp ← הגדרות ← מכשירים מקושרים ← קישור מכשיר*.
4. כשהערוץ "מחובר" (ירוק) — שומרים את ה-**API Token** של הערוץ.

### שלב 2 — הגדרת הסודות ב-Supabase (~5 דקות)

בטרמינל, בתיקיית הפרויקט:

```bash
# הטוקן מ-Whapi (שלב 1)
supabase secrets set WHAPI_TOKEN="<הטוקן-מ-Whapi>"

# סוד משותף שממציאים — שומר שרק Whapi יוכל לדבר עם המערכת
supabase secrets set GROUP_WEBHOOK_SECRET="<מחרוזת-אקראית-ארוכה>"
```

(ליצירת מחרוזת אקראית: `openssl rand -hex 24`)

### שלב 3 — הפניית Whapi אל המערכת (~5 דקות)

בלוח הבקרה של Whapi ← Settings ← **Webhooks**:

- **URL**:
  ```
  https://tgropjisnheppsxejfdn.supabase.co/functions/v1/whatsapp-group-webhook?secret=<אותו-סוד-משלב-2>
  ```
- **Events**: מסמנים `messages` (הודעות נכנסות) בלבד.
- **Method**: POST.

### שלב 4 — הדלקה במסך ההגדרות (~1 דקה)

*הגדרות מערכת ← WhatsApp ← בוט בקבוצות*:
1. מדליקים את **"הפעלת הבוט בקבוצות"**.
2. מוודאים שמילת ההפעלה נכונה (ברירת מחדל: `גרפיקה`).
3. שמירת הגדרות.

### שלב 5 — הוספת הבוט לקבוצה (~1 דקה)

1. פותחים את קבוצת ה-WhatsApp הרצויה (או יוצרים קבוצת בדיקה).
2. *פרטי קבוצה ← הוספת משתתף* ← בוחרים את **המספר החדש של הבוט**.
3. מומלץ להודיע לחברי הקבוצה שיש בוט שקורא את ההודעות (שקיפות ופרטיות).

### שלב 6 — בדיקת קצה־לקצה (~10 דקות)

מריצים בקבוצה, לפי הסדר:

| # | שולחים בקבוצה | מצפים |
|---|---|---|
| 1 | `בוקר טוב לכולם` | הבוט **שותק** |
| 2 | `גרפיקה` | הבוט עונה עם התפריט הראשי |
| 3 | `1` → ברייף לתמונה | הבוט אוסף ברייף ומפיק — **התמונה חוזרת לקבוצה** |
| 4 | אדם אחר עונה לתפריט: `1` (בלי מילת הפעלה) | הבוט **ממשיך את הסשן המשותף** |
| 5 | `גרפיקה התחל מחדש` | השיחה של אותו שולח מתאפסת לתפריט |

אם משהו לא מגיב — בודקים ב: *Supabase ← Edge Functions ← whatsapp-group-webhook ← Logs*,
ובטבלת ה-logs במערכת (חיפוש `group_message_triggered` / `group_webhook_bad_secret`).

### שלב 7 — כיבוי חירום

בכל רגע: *הגדרות ← WhatsApp ← בוט בקבוצות ← כיבוי המתג* ← הבוט מפסיק לענות
בקבוצות מיידית (ההודעות פשוט מסוננות בכניסה). אפשר גם להוציא את המספר מהקבוצה.

## מגבלות ידועות בפיילוט (מה עוד לא נתמך בקבוצות)

- **קבצים נכנסים**: הבוט קורא רק טקסט מהקבוצה (כולל כיתוב על תמונה). העלאת
  קבצים כחומרי ברייף — דרך הצ'אט הפרטי או האתר. (שלב המשך אם יידרש.)
- **כפתורים אינטראקטיביים**: בקבוצות נשלח תמיד תפריט טקסט ממוספר (1/2/3) —
  מגבלת הפלטפורמה.
- **זיהוי משתמשים**: שולח בקבוצה מזוהה לפי הטלפון שלו מול משתמשי האתר, כמו
  בצ'אט פרטי. מי שלא רשום יקבל הזמנה להירשם (בתוך הקבוצה, מוגבל פעם ב-6 שעות).
- **סיכון חסימה**: המסלול הלא־רשמי מסכן את מספר הבוט. לכן מספר ייעודי בלבד.
