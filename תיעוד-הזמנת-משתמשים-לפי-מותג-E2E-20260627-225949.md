# תיעוד E2E — הזמנת משתמשים חדשים לפי מותג

תאריך: 2026-06-27 · חותמת: 20260627-225949

פיצ'ר: המנהל מנפיק קישור הרשמה ייעודי למותג. מי שנרשם דרכו רואה את הלוגו והשפה
העיצובית של המותג, ועם ההרשמה משויך אוטומטית למותג ויכול להתחיל לעבוד מיד — בלי
שמנהל צריך לשייך ידנית.

---

## החלטות מוצר (אושרו ע״י המשתמש)
1. **קישור רב-שימושי + ביטול** — קישור אחד למותג, כמה אנשים יכולים להירשם דרכו, עם מונה נרשמים ואפשרות ביטול/הפעלה מחדש.
2. **הרשמה נשארת פתוחה** — `/signup` ללא token ממשיך ליצור משתמש רגיל (ללא מותג). בנוסף נוסף כפתור בהגדרות להסתרת/הצגת הקישור "הירשם" במסך הכניסה.
3. **ללא תוקף** — הקישור תקף עד שהמנהל מבטל אותו.

---

## ארכיטקטורה קיימת (רקע)
- `profiles` (role admin/user, `can_create_outputs`), `user_brands` (שיוך משתמש↔מותג), `brands` (logo_path, color_palette).
- הרשמה דרך edge function `signup` (יוצר משתמש `user` עם `can_create_outputs=false`, ללא מותג). טריגר `handle_new_user` מקים את ה‑profile.
- `useBrandTheme` צובע את האפליקציה לפי המותג היחיד של המשתמש המחובר.

## הזרימה המלאה
1. מנהל → משתמשים והרשאות → טאב "הזמנות" → בוחר מותג → "צור קישור" (מועתק אוטומטית).
2. שולח את הקישור `${origin}/signup?invite=<token>` ללקוח.
3. הלקוח פותח → רואה לוגו + צבעי המותג → נרשם.
4. נכנס ישר מוכן לעבוד על המותג (משויך + `can_create_outputs`), `useBrandTheme` צובע את האפליקציה.
