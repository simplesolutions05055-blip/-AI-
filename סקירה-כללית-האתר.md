## 🔴 אבטחה — הדברים החשובים ביותר

### 1. **`process-request` ו-`send-output` פתוחים לאינטרנט**

**הבעיה:**  
שתי הפונקציות מוגדרות `verify_jwt = false` ב-[supabase/config.toml](supabase/config.toml) ואין בהן שום בדיקת אימות או סוד פנימי:
- [`process-request/index.ts`](supabase/functions/process-request/index.ts) — מריצה את כל צינור הג'נרציה (עלות OpenAI)
- [`send-output/index.ts`](supabase/functions/send-output/index.ts) — שולחת מיילים דרך Resend

**הסיכון:**  
מי שמשיג `request_id` (UUID) יכול:
- להריץ ג'נרציה שוב ושוב — עלות חוזרת ב-OpenAI
- לשלוח מיילים לעצמו או למישהו אחר

UUID קשה לנחש, אך מופיע בלוגים, מיילים, URLs שמשתמשים משתפים או חושפים.

**הפתרון (מהיר):**
- תוודא: האם יש שירות חיצוני (worker בVercel?) שקורא להן בלי טוקן? 
  - אם **לא**: הדליקו `verify_jwt = true` ב-config.toml — מוגן.
  - אם **כן**: הוסיפו בדיקת סוד בתוך הפונקציה, כמו ב-[conversation-maintenance](supabase/functions/conversation-maintenance/index.ts:12) שבודקת `CRON_SECRET`.

### 3. **`openai_key` override — race condition**

**הקובץ:** [`process-request/index.ts:33-42`](supabase/functions/process-request/index.ts:33)

**הבעיה:**
```typescript
const overrideKey = ... // מפתח ה-override

  await processRequest(request_id);
  // ...

```

**הפתרון:**
העבירו את ה-override כפרמטר לאורך הצינור במקום global state.

## ⚡ ביצועים

### 1. **אין Code-Splitting לראוטים**

**הקובץ:** [`src/App.tsx:1-22`](src/App.tsx)

**הבעיה:**
כל 20+ הדפים מיובאים eagerly:
```typescript
import DashboardPage from '@/pages/admin/DashboardPage';
import RequestsCostsPage from '@/pages/admin/RequestsCostsPage';
// ... עוד 18 דפים
```

**התוצאה:**

**סיכון PWA:**
ה-precache מוגבל ל-3MB (ב-[vite.config.ts:41](vite.config.ts:41)). אתם כבר **מעל הגבול**.

**הפתרון:**
```typescript
const DashboardPage = lazy(() => import('@/pages/admin/DashboardPage'));
const RequestsCostsPage = lazy(() => import('@/pages/admin/RequestsCostsPage'));
// ... לכל דף
```

זה יקצר את הטעינה הראשונית דרמטית ויפתור את בעיית ה-precache.

### 3. **`useProfile` לא מאזין ל-`onAuthStateChange`**

**הבעיה:**
אם הסשן פג במהלך עבודה (token expired), המשתמש **לא יופנה ללוגין** עד רענון דף. פעולות פשוט ייכשלו בשקט.

**מה בעד זה:**
[`usePwaInstall.ts:83`](src/components/pwa/usePwaInstall.ts:83) כן משתמש ב-`onAuthStateChange`.

**הפתרון:**
הוסיפו listener ב-`useProfile`:
```typescript
useEffect(() => {
  const { data: listener } = db.auth.onAuthStateChange((event, session) => {
    if (!session) {
      setState({ loading: false, profile: null, hasBrand: false, requireUploads: false });
    }
  });
  return () => listener?.subscription.unsubscribe();
}, []);
```

## 🏗️ ארכיטקטורה — חוזקות

### ✅ מה עובד בצורה טובה:

1. **RLS + Auth:** מופעל על כל הטבלאות עם מדיניות ברורה
2. **Twilio Webhook:** אימות חתימה, idempotency לפי MessageSid, rate-limit
3. **Shared Logic:** `handleInbound` משותפת לסימולטור ול-WhatsApp — זה טוב
4. **Service Role Isolation:** כל write עובר דרך functions, לא ישירות לקלায
5. **Debounce + Locks:** הודעות רצופות מתמוזגות, requests מנעולים עם `locked_at`
