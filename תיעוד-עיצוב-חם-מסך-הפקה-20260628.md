# תיעוד סשן — מעבר לעיצוב PrimeOS נקי (יציאה מהמראה הדיפולטי של AI)

**תאריך:** 2026-06-28
**מטרה:** להוציא את ממשק האדמין מהמראה הגנרי של "AI default" (כחול רווי על רקע קריר, פינות בועתיות, צללים רכים, badges פסטל), אבל בלי להתנתק מהמותג. הכיוון החם/מותגי היה שלב ביניים טוב, אך לאחר בדיקת לוגו PrimeOS הוחלט לעבור ל‑**PrimeOS enterprise נקי**: navy / blue / teal / green על משטחים שקטים.

**החלטות שננעלו בסשן:**
- כיוון עדכני: **PrimeOS enterprise נקי** — מבוסס על הלוגו: נייבי כהה, כחול, ירוק וטורקיז, בלי לחזור ל‑AI/SaaS כחול גנרי.
- הכיוון הקודם **חם / מותגי** נשאר כתוב בהיסטוריה, אבל כבר לא הכיוון הסופי למותג PrimeOS.
- היקף: **פיילוט על מסך אחד** — `/admin/production` (מסך "הפקת תוצרים", הקומפוננטה `ProductionPicker`).
- עיקרון מנחה: **ה‑brand accent נשאר דינמי per-tenant** ([useBrandTheme.ts](src/lib/useBrandTheme.ts)). ברירת המחדל כעת PrimeOS navy/blue; מותג אמיתי עדיין יכול לדרוס את ה‑accent.

## החלטה חדשה בעקבות הלוגו

הלוגו של PrimeOS מדבר בשפה של:
- כחול טכנולוגי חזק
- נייבי כהה
- ירוק חי
- טורקיז/מעבר כחול־ירוק
- תחושה של מערכת, מהירות, אופטימיזציה ו‑SaaS תפעולי

המסקנה: הפלטה החמה `terra / cream / parchment` הוציאה אותנו מהמראה הגנרי של AI, אבל לא הייתה מספיק PrimeOS. לכן הוחלט לשמור את עקרונות ה‑UI שכבר עבדו — פחות glow, פחות pill, פחות shadow, primary נדיר, אייקונים מוכרים מ‑Lucide — ולהחליף את הצבעים לפלטה מותגית נקייה.

העיקרון החדש: **לא לצבוע את כל הממשק בכחול/ירוק.** הכחול הוא צבע מערכת/פעולה, הירוק נשאר highlight/הצלחה/צמיחה, והמשטחים נשארים כמעט לבנים כדי שהמוצר ירגיש כלי עבודה ולא landing page.

## פלטת PrimeOS החדשה

מוגדרת ב‑[src/index.css](src/index.css) בתוך `.theme-warm` משיקולי תאימות לקוד הקיים. השם נשאר זמנית, אבל התוכן כבר PrimeOS:

```css
.theme-warm {
  --bg-page: #f6f9f8;
  --bg-surface: #ffffff;
  --bg-subtle: #edf4f2;
  --surface-2: #f3f7f6;
  --text-strong: #071a33;

  --text-faint: #8da0ad;
  --border-warm: #d7e3e0;
  --border-soft: #e6eeee;
  --warm-accent: #0b4f9f;
  --warm-accent-dark: #071a33;
  --warm-accent-soft: #e7f1fb;
  --warm-nudge-rgb: 18 104 216;
  --tint-clay: #e8f2ff;
  --tint-sand: #eaf7f5;
  --tint-olive: #e9f8ee;
  --tint-ochre: #edf4ff;
}
```

Tailwind `brand` fallback עודכן ב‑[tailwind.config.ts](tailwind.config.ts):

```ts
brand: {
  DEFAULT: 'rgb(var(--brand-rgb, 11 79 159) / <alpha-value>)',
  dark: 'rgb(var(--brand-dark-rgb, 7 26 51) / <alpha-value>)',
}
```

⚠️ שינוי `tailwind.config.ts` דורש restart מלא ל‑`npm run dev`.

## מה זה ה"מראה של AI" (האבחנה)

הכחול הוא רק חלק. הצירוף שמייצר את התחושה:
1. accent כחול רווי (`#1f6feb`) על רקע קריר
2. wash כחול/ירוק רדיאלי ברקע הגלובלי
3. פינות עגולות מאוד (`rounded-2xl`/`rounded-full`)
4. badges פסטל "סוכרייה"
5. צללים רכים כבדים + hover-lift בועתי
6. הרבה כפתורים מלאים (כל אחד "צועק primary")

המדריך מזהיר: **שינוי צבע בלבד = עדיין גנרי.** צריך לשנות את כל המערכת.

## כל הקבצים ששונו

### 1. [tailwind.config.ts](tailwind.config.ts)
ה‑fallback של `brand` שונה מכחול לטרהקוטה (משפיע רק כשאין מותג יחיד משויך):
```ts
brand: {
  DEFAULT: 'rgb(var(--brand-rgb, 165 76 43) / <alpha-value>)',   // היה 31 111 235
  dark:    'rgb(var(--brand-dark-rgb, 142 66 37) / <alpha-value>)', // היה 26 95 206
}
```
⚠️ **שינוי ב‑tailwind.config.ts דורש restart ל‑`npm run dev`** (HMR לא מספיק).

### 2. [src/index.css](src/index.css)
- **נוסף** בלוק `.theme-warm` (כל הטוקנים למעלה).
- **הוסר ה‑wash הגלובלי:** ה‑`background` של `html,body` היה גרדיאנט רדיאלי כחול+ירוק → עכשיו רקע שטוח חם `#f7f5f1`. **משפיע על כל המסכים.**
- 9 ה‑fallbacks הכחולים באנימציות (`pick-me-glow`, `revise-scan`, `revise-sheen`) שונו מ‑`31 111 235` ל‑`165 76 43`.

### 3. [src/pages/admin/ProductionPage.tsx](src/pages/admin/ProductionPage.tsx)
**קבועים משותפים (שורות ~82-88)** — משפיעים גם על מסכי ה‑flow (`/admin/production/:type`):
- `PANEL`: `rounded-2xl` → `rounded-xl`, הוסר הצל הרך הכבד `0_20px_50px`.
- `PRIMARY_BUTTON`: **תוקן באג** — הצל הכחול הקשיח `rgba(59,130,246)` (שזהר כחול בלי קשר ל‑brand) הוסר; הוסר ה‑hover-lift הבועתי; `rounded-xl` → `rounded-lg`; `hover:bg-brand-dark`.
- `SECONDARY_BUTTON`: `rounded-xl` → `rounded-lg`, הוסר hover-lift.
- `CHIP_BUTTON`: `rounded-full` → `rounded-lg`.

**`ProductionPicker` (הקומפוננטה של המסך, שורות ~560-1092):**
- שורש עטוף ב‑`className="theme-warm ..."` + רקע `bg-[var(--bg-page)]`.
- כל ה‑hex הקרים (`#F0F2F7`, `#1A1A2E`, `#94A3B8`, `#CBD5E1`, `#EAECF0`, `#E5E7EB`, `#F1F5F9`, `#F8FAFC`, `#64748B`, `#334155`) → טוקנים חמים.
- `pickerChips` colors: פסטל (`#FEE2E2` וכו') → `var(--tint-*)`.
- `recentOutputCategories` bg: פסטל → `var(--tint-*)`.
- chips של בחירת סוג: `rounded-full` → `rounded-[10px]`, גבולות חמים (active נשאר brand).
- chips של מותגים: `rounded-full` → `rounded-[10px]`, גבולות חמים.
- textarea: רקע `var(--surface-2)`, גבול חם, `rounded-[10px]`.
- תיבות error → טוקני danger חמים; תיבת hint → טוקני warning חמים.
- **כפתור "צור תוצר" (primary):** active = `bg-brand` מלא + `hover:bg-brand-dark`; **disabled = אפור־חם ניטרלי** (`bg-[var(--bg-subtle)]` + גבול) במקום טרהקוטה דהוי.
- **4 כפתורי כרטיסי התוצרים:** היו מלאים `bg-brand` → **ghost** (מתאר `border-brand/35`, שקוף, טקסט "יצירה"/"צפייה", מתמלא ב‑hover).
- כותרת H1: `text-[26px] font-bold` → `text-[30px] font-extrabold tracking-tight`.

## מערכת הכפתורים שנוצרה (3 רמות מובחנות)

| רמה | דוגמה | עיצוב |
|---|---|---|
| **Primary** (נדיר) | "צור תוצר" | טרהקוטה מלא, hover מתכהה, disabled אפור ניטרלי |
| **Secondary** | "העלה קובץ" | cream + גבול חם |
| **Tertiary/Ghost** | כפתורי כרטיסי תוצרים, "כל התוצרים" | מתאר שקוף, מתמלא ב‑hover |

עיקרון: primary אחד אמיתי, השאר שקטים (לפי המדריך — "primary צריך להיות נדיר").

## איך להמשיך בסשן חדש (הוראות מעשיות)

1. **להריץ:** `npm run dev` (פורט 8080). אם שינית `tailwind.config.ts` — **restart מלא**, לא רק רענון.
2. **הדפוס שעבד:** עוטפים מסך ב‑`<div className="theme-warm ...">`, ואז כל צבע = `bg-[var(--bg-surface)]` / `text-[var(--text-strong)]` / `border-[var(--border-warm)]` וכו'. ה‑accent נשאר `bg-brand` / `text-brand` (דינמי).
3. **לא לגעת** ב‑`useBrandTheme.ts` ובלוגיקת ה‑brand הדינמי.
4. **ההמלצה הבאה הכי מורגשת:** לחמם את ה‑**shell** (סיידבר ב‑[AdminNav.tsx](src/components/AdminNav.tsx) + [AdminLayout.tsx](src/pages/admin/AdminLayout.tsx)) — להעביר את הפאנל מ‑`bg-white`/`border-[var(--border)]` קרים לבז' חם.
5. **אימות:** המסך מאחורי login, אז `preview` נקי לא יראה אותו — בודקים חי על :8080. תמיד להריץ `npx tsc --noEmit` אחרי שינויים.
