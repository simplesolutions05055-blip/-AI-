## 1. מטרת המסמך

מסמך זה מרכז את כל הדרישות שעלו בשתי שיחות האפיון, את ההחלטות שכבר התקבלו ואת הנושאים שעדיין דורשים תשובה. לאחר מילוי ואישור המסמך ניתן יהיה לסגור את ה־System Message, את מבנה השיחה, את סוגי התוצרים ואת קריטריוני הקבלה של ה־MVP.

---

## 2. תקציר עסקי

המטרה היא להקים **סוכן AI ארגוני הזמין דרך WhatsApp**. משתמש שולח בקשה בשפה חופשית למספר ייעודי. המערכת מנהלת איתו שיחה קצרה, אוספת פרטים חסרים וכתובת מייל, מעבדת את הבקשה באמצעות AI, מייצרת תוצר, מבצעת בדיקות איכות בסיסיות ומציגה אותו למנהל לאישור. רק לאחר אישור מנהל התוצר נשלח לכתובת המייל שנמסרה.

### דוגמת שימוש שעלתה בשיחה

ארגון או עירייה צריכים בשעת לילה מאוחרת להפיק חומר דחוף לתורמים, משקיעים או קהל יעד אחר. במקום להמתין לצוות אנושי, נציג שולח בריף ב־WhatsApp ומקבל תוצר ראשוני במייל לאחר בדיקת מנהל.

## 3. מטרות ה־MVP

1. לקבל הודעות WhatsApp באמצעות Twilio.
2. לקשר מספר הודעות לאותה שיחה ובקשה.
3. לזהות מידע חסר ולשאול שאלות המשך.
4. לאסוף כתובת מייל תקינה.
5. להפוך את השיחה לבריף מובנה.
6. לשלוח את הבריף ל־OpenAI.
7. להפיק טקסט, תמונה ו/או PDF בהתאם לסוג הבקשה שאושר.
8. לבצע עד מספר מוגדר של ניסיונות שיפור ובדיקות QA.
9. להציג שיחות, תוצרים, סטטוסים, לוגים ועלויות בממשק ניהול.
10. לאפשר אישור, דחייה, עריכה ויצירה מחדש.
11. לשלוח תוצר מאושר במייל דרך Resend.
12. לעדכן את המשתמש ב־WhatsApp שהתוצר נשלח.
13. לפעול 24/7 ברמת קליטת בקשות, בכפוף לזמינות ספקי צד שלישי.

## 4. החלטות שכבר התקבלו

- שם המוצר בשלב זה: **סוכן AI ארגוני דרך WhatsApp**.
- WhatsApp יחובר דרך **Twilio WhatsApp API**.
- ה־Frontend וממשק הניהול ייבנו ב־**React/TypeScript**.
- ה־Frontend וה־API ייפרסו ב־**Vercel**.
- מסד הנתונים, Auth ואחסון הקבצים יהיו ב־**Supabase**.
- עיבוד AI יתבצע דרך **OpenAI API**.
- יצירת תמונה תתחיל עם מודל התמונה של OpenAI; אם האיכות אינה מספקת, תיבדק חלופה של **Gemini/Nano Banana**.
- שליחת מיילים תתבצע דרך **Resend**.
- ב־MVP יהיה **מנהל יחיד**.
- אין פרסום או תזמון פוסטים לרשתות חברתיות ב־MVP.
- אין מערכת משתמשים והרשאות מורכבת.
- אין ריבוי לקוחות/ארגונים במערכת אחת.
- שום תוצר אינו נשלח למייל ללא אישור מנהל.
- המחיר שסוכם בשיחה: **5,000 ₪**, לא כולל עלויות ספקים.
- אבני תשלום שסוכמו:
  - 2,000 ₪ בתחילת העבודה.
  - 1,000 ₪ לאחר תוצר ראשון מקצה לקצה: בקשה ב־WhatsApp שמגיעה כתוצר למייל.
  - 2,000 ₪ במסירת הפרויקט.

## 15. ארכיטקטורה

```text
WhatsApp User
    ↓
Twilio WhatsApp API
    ↓
Vercel API Webhook
    ↓
Supabase Database / Storage
    ↓
Async processing / Queue
    ↓
OpenAI / Gemini
    ↓
Supabase outputs
    ↓
React Admin Dashboard on Vercel
    ↓
Manager approval
    ↓
Resend
    ↓
Customer email + WhatsApp confirmation
```

### 15.1 עיבוד אסינכרוני

Webhook חייב להשיב במהירות ל־Twilio ולא להמתין לסיום AI.

ה־Webhook:

1. מאמת.
2. שומר הודעה.
3. מחזיר 200.
4. מפעיל משימת עיבוד נפרדת.

יש לבחור מנגנון Queue/Workflow שמתאים למגבלות Vercel. אם עיבוד התמונות או PDF ארוך מדי, ניתן להעביר רק את משימת העיבוד ל־Supabase Edge Function או לשירות Worker ייעודי.

## 18. ביצועים וזמינות

- קליטת הודעות: 24/7 בכפוף לזמינות Twilio/Vercel/Supabase.
- אישור קבלה: בתוך שניות.
- שאלת המשך: יעד של עד 30 שניות.
- תוצר טקסטואלי: יעד של מספר דקות.
- תמונה/PDF: בהתאם לזמן ספק ה־AI.
- שליחת מייל לאחר אישור: יעד של עד דקה.

הערכה שנדונה: 50 משתמשים ביום ועד 50 הודעות למשתמש. יש לאשר האם זו תחזית אמיתית או תרחיש קיצון.
