# תיקון רווח לבן במסך מיתוג במובייל

תאריך: 2026-06-30

## תקציר

במסך `מיתוג`, במובייל, אחרי בחירת מותג מהרשימה המערכת גללה אוטומטית לטופס העריכה. לאחר הגלילה הופיע רווח לבן גדול בחלק העליון של כרטיס העריכה, מעל הכותרת `עריכת מקום`. הרווח לא הופיע במצב המקורי ונעלם אחרי גלילה ידנית קטנה.

## גורם

ב־`src/pages/admin/BrandingPage.tsx`, כותרת כרטיס העריכה הוגדרה כ־`sticky` עם offset עליון:

```tsx
top-[calc(var(--safe-top)+3.75rem)]
```

ה־offset הזה התאים למצב שבו יש header קבוע מעל התוכן, אבל במסך המובייל של ה־admin layout אין header עליון קבוע. לכן, אחרי `scrollIntoView({ block: 'start' })`, הכותרת הדביקה נדחפה מטה ונוצר רווח לבן בתוך כרטיס העריכה.

## תיקון

עודכן `src/pages/admin/BrandingPage.tsx`:

- לכרטיס העריכה נוסף `scroll-mt-4`, כדי לתת מרווח קטן ונקי כשגוללים אליו.
- ה־sticky header של כרטיס העריכה שונה מ־`top-[calc(var(--safe-top)+3.75rem)]` ל־`top-0`.

הקוד לאחר התיקון:

```tsx
<section
  id="brand-editor"
  className="scroll-mt-4 min-w-0 space-y-4 rounded-xl border border-[var(--border)] bg-white p-4 shadow-[0_1px_2px_rgba(15,23,42,0.04)]"
>
  <div className="sticky top-0 z-20 -mx-4 -mt-4 flex flex-col gap-2 border-b border-[var(--border)] bg-white p-4 sm:static sm:mx-0 sm:mt-0 sm:flex-row sm:items-center sm:justify-between sm:border-0 sm:bg-transparent sm:p-0">
```

## תוצאה צפויה

במובייל, בחירת מותג מהרשימה עדיין גוללת אל טופס העריכה, אבל ללא הרווח הלבן המיותר מעל `עריכת מקום`.

## בדיקות

```bash
pnpm run typecheck
```
