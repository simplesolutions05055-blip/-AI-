# תיעוד סשן — סגירת באנר אונבורדינג ועמוד הגדרות למשתמש רגיל

**תאריך:** 2026-06-28  
**מטרה:** לאפשר למשתמש רגיל לסגור לצמיתות את באנר השלמת האונבורדינג, לשמור את ההחלטה ברמת בסיס הנתונים, ולהוסיף עמוד הגדרות נגיש מהתפריט עם מעבר לעמוד האונבורדינג.

---

## 1. הרקע

היה באנר עליון שמופיע למשתמש רגיל שלא השלים את הגדרת המותג:

```html
<div class="flex items-center gap-3 border-b border-amber-200 bg-amber-50 px-4 py-2 text-sm text-amber-900">
  <span class="flex-1">עוד לא השלמתם את הגדרת המותג — מומלץ להוסיף מסמכים וקבצים.</span>
  <button class="shrink-0 rounded-md bg-brand px-3 py-1 text-xs font-semibold text-white hover:bg-brand-dark">להשלמה</button>
  <button aria-label="סגירה" class="shrink-0 text-amber-700 hover:text-amber-900">✕</button>
</div>
```

לפני השינוי, כפתור הסגירה הסתיר את הבאנר רק בסשן הנוכחי באמצעות state מקומי ב-React. אחרי רענון או כניסה מחדש הבאנר חזר.

---

## 2. החלטת מימוש

הסגירה נשמרת בתוך `profiles.onboarding` כשדה JSONB:

```json
{
  "banner_dismissed_at": "2026-06-28T..."
}
```

הסיבה לבחירה הזו:

- `profiles.onboarding` כבר מחזיק state פר-משתמש של האונבורדינג.
- קיימת כבר מדיניות RLS שמאפשרת למשתמש לעדכן את הפרופיל שלו.
- טריגר `guard_profile_self_update` כבר מגן מפני שינוי שדות רגישים כמו `role`, `can_create_outputs`, `email`, ו-`id`.
- אין צורך בטבלה חדשה עבור preference אחד.

---

## 3. שינויי DB

נוספה מיגרציה:

```text
supabase/migrations/20260628113000_onboarding_banner_dismissal.sql
```

המיגרציה מתעדת את המבנה החדש של `profiles.onboarding`:

```sql
comment on column public.profiles.onboarding is
  'Per-user onboarding state: details_done/docs_done/files_done/hard_completed_at/banner_dismissed_at.';
```

לא נוספה עמודה חדשה, כי השדה נשמר בתוך JSONB קיים.

## 4. שינויי קוד

### `src/lib/useProfile.ts`

עודכן הטיפוס `OnboardingState`:

```ts
banner_dismissed_at?: string | null;
```

### `src/lib/onboarding.ts`

עודכנה לוגיקת הצגת הבאנר:

- אדמין עדיין לא רואה באנר.
- אם `profile.onboarding.banner_dismissed_at` קיים, הבאנר לא מוצג.
- אם אין dismiss, נשמרת הלוגיקה הקודמת: הבאנר מוצג רק כשהמשתמש עבר את החסם אבל לא השלים את כל השלבים האופציונליים.

### `src/components/OnboardingBanner.tsx`

הבאנר מקבל עכשיו:

- `userId`
- `onboarding`

בלחיצה על ✕:

1. הבאנר מוסתר אופטימית במסך.
2. מתבצע `update` ל-`profiles.onboarding` עם `banner_dismissed_at`.
3. אם העדכון נכשל, הבאנר חוזר להיות מוצג.

## 5. עמוד הגדרות למשתמש רגיל

נוסף עמוד חדש:

```text
src/pages/admin/UserSettingsPage.tsx
```

העמוד נמצא בנתיב:

```text
/admin/user-settings
```

העמוד כולל:

- כותרת "הגדרות".
- הסבר קצר על ניהול פרטים והשלמת הגדרת המותג.
- כרטיס "אונבורדינג".
- כפתור "מעבר לאונבורדינג" שמוביל ל-`/onboarding`.

---

## 6. תפריט והרשאות נתיבים

### `src/components/AdminNav.tsx`

נוסף item חדש לתפריט תחת "הגדרות":

```ts
{ href: '/admin/user-settings', label: 'הגדרות', userOnly: true, icon: 'userSettings' }
```

נוסף `userOnly` ל-`NavLink`, כך שהקישור מופיע רק למשתמש רגיל ולא לאדמין.

נוסף icon:

```ts
UserCog
```

בניווט התחתון במובייל למשתמש רגיל נוספה הגדרות לצד:

- הפקה
- תוצרים
- הגדרות

### `src/pages/admin/AdminLayout.tsx`

עודכן whitelist של משתמש רגיל:

```ts
const USER_ALLOWED_PREFIXES = [
  '/admin/production',
  '/admin/quote',
  '/admin/files',
  '/admin/user-settings',
];
```

כך שמשתמש רגיל יכול לגשת לעמוד ההגדרות החדש.

### `src/App.tsx`

נוסף route:

```tsx
<Route path="user-settings" element={<UserSettingsPage />} />
```

## 7. אימות

הורצו בדיקות:

```bash
npm run build
npm run typecheck
```

שתיהן עברו בהצלחה.

במהלך `build` הופיעו רק אזהרות קיימות של Vite:

- CJS build של Vite Node API deprecated.
- chunks גדולים.
- הערת dynamic/static import על `jspdf`.

לא היו שגיאות קומפילציה או TypeScript.
