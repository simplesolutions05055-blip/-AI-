# תיעוד סשן — טקסט פוסט אוטומטי לכל תמונה + תצוגת פייסבוק בתזמון (2026-07-02)

## מטרת הסשן

1. כשמייצרים **תמונה/גרפיקה**, המערכת תפיק אוטומטית גם את **טקסט הפוסט** המוכן לפרסום, ותציג אותו מתחת לתמונה — מקצה לקצה (צנרת, UI, וואטסאפ).
2. אפשרות **לערוך את הטקסט עם AI** (כמו תיקוני התמונה: כותבים מה לשנות ומקבלים גרסה מעודכנת).
3. במודאל תזמון לרשתות: **צירוף אוטומטי של תמונת ה-AI** לפוסט + **כפתור "דוגמה של הפוסט"** שפותח תצוגה מקדימה בעיצוב פייסבוק אמיתי (RTL, מלל למעלה, תמונה מתחת).

> חלק מהעבודה בוצע במקביל עם Codex; Claude אימת את הכל, תיקן פער אחד (הזרקת `brand_id` לבריף ב-ResultCard), ופרס מחדש את כל הפונקציות.

## 1. צד שרת — טקסט פוסט אוטומטי בצנרת (E2E)

### `supabase/functions/_shared/worker.ts`
- **יצירת תמונה** (`generateAndQa`, ענף `image`): מיד אחרי העלאת התמונה ל-storage נקראת `generateSocialCaption(brief, 'facebook')` והטקסט נשמר על התוצר ב-`outputs.text_content`.
  - Best-effort: כשל ביצירת הטקסט לא מפיל את התמונה (נרשם `social_caption_failed` כאזהרה בלבד).
  - העלות נרשמת ב-`usage_events` (recordUsage + estimateTextCost).
- **משלוח וואטסאפ** (`deliverContentToWhatsApp`): לתמונות בלבד, אחרי המדיה נשלחת הודעת המשך `"טקסט מוכן לפוסט:\n\n<הפוסט>"` — בכל המסלולים (סימולטור, אמיתי, fallback ללא URL). ל-PDF **לא** נשלח (שם `text_content` הוא גוף המסמך).
  - שיחות `production-form` לא שולחות כלום (כמו קודם) — ה-UI קורא ישירות מהתוצר.

### `supabase/functions/_shared/openai.ts`
- `generateSocialCaption` קיבלה פרמטר רביעי אופציונלי `revision: { currentCaption, feedback }` — מצב עריכה: המודל מקבל את הפוסט הקיים + בקשת השינוי ומחזיר גרסה מעודכנת בלבד, עם אותם גבולות (לא להמציא עובדות).

### `supabase/functions/generate-presentation/index.ts` — מצב `social_caption` מורחב
שלושה תתי-מצבים:
| מצב | פרמטרים | התנהגות |
|---|---|---|
| יצירה | `brief` (+ `platform`) | כותב פוסט חדש מהבריף + המוח העסקי של המותג |
| עריכת AI | `current_caption` + `feedback` | משכתב את הפוסט לפי המשוב |
| שמירה ידנית | `save_only: true` + `current_caption` | שומר בלבד, בלי קריאת AI |

- פרמטר `output_id`: כשנשלח, התוצאה נשמרת על שורת ה-outputs (`text_content`, מוגבל ל-`output_type='image'`) — ההתמדה נעשית בצד שרת (service role), בלי תלות ב-RLS של הקליינט.
- לוגים: `social_caption_generated` / `social_caption_revised` / `social_caption_saved`.

### `supabase/functions/edit-image/index.ts`
- עריכת תמונה (img2img) **מעבירה את טקסט הפוסט הלאה**: `text_content` מועתק מהתוצר המקורי לתוצר החדש (עריכת פיקסלים לא משנה את המסר).
- התגובה מחזירה עכשיו גם `output_id` ו-`text_content` כדי שה-UI ימשיך לערוך את הטקסט על השורה החדשה.

## 2. צד לקוח — הצגה ועריכה

### `src/lib/social.ts`
- `fetchSocialCaption(...)` — פרמטר חמישי `outputId` להתמדה בצד שרת.
- חדש: `reviseSocialCaption(currentCaption, feedback, brief, requestId, outputId)` — עריכת AI.
- חדש: `saveSocialCaption(caption, outputId, requestId)` — שמירה ידנית בלבד.

### `src/pages/admin/RevisePage.tsx` (מסך התוצאה בפועל לתמונות)
- טעינת התוצר כוללת `id, text_content`; מוצג פאנל **"טקסט הפוסט"** מתחת לתמונה:
  - textarea הניתן לעריכה ישירה (שמירה אוטומטית ב-blur דרך `saveSocialCaption`).
  - כפתור העתקה.
  - שורת עריכת AI: "מה לשנות בטקסט?" + כפתור "עדכון עם AI" (Enter מפעיל), עם אנימציית הכתיבה (`social-caption-writing/scan`).
- **תוצרים ישנים** (נוצרו לפני הפיצ'ר): הטקסט נכתב אוטומטית בכניסה הראשונה לעמוד ונשמר על התוצר (guard נגד StrictMode double-mount, retry בכשל).
- אחרי **עריכת תמונה**: `postOutputId` מוסט לתוצר החדש (הטקסט הועתק בצד שרת).
- אחרי **רגנרציה מלאה**: הטקסט הטרי מהצנרת נטען מהתוצר החדש.
- מודאל התזמון מקבל `captionSource={kind:'text'}` עם הטקסט הנוכחי — בלי חיוב AI כפול.

### `src/pages/admin/ProductionPage.tsx` (ResultCard — מסך תוצאה בהפקה)
- אותו פאנל "טקסט הפוסט" (הוסף ע"י Codex); Claude תיקן: `postBrief()` מזריק את `brand_id` לבריף כדי שהטקסט יתבסס על המוח העסקי של המותג.

### `src/components/SocialScheduleSection.tsx`
- סנכרון: כשהטקסט בעמוד משתנה (עריכת AI) והמודאל סגור — הכיתוב במודאל מתעדכן; עריכות בתוך המודאל לא נדרסות.

## 3. מודאל התזמון — צירוף תמונת ה-AI + תצוגת פייסבוק

### צירוף אוטומטי של התמונה (`SocialScheduleSection.tsx`)
- prop חדש `producedImage: { url, storagePath }` — מועבר מ-RevisePage (כולל אחרי עריכה/רגנרציה — `result.storagePath` נוסף ל-state) ומ-ResultCard בעמוד ההפקה.
- בפתיחת המודאל התמונה נכנסת אוטומטית ל"מדיה לפרסום" עם:
  - תג "נוצרה עם AI" על התמונה הממוזערת (`MediaItem.aiGenerated`).
  - הודעה סגולה: *"התמונה שיצרנו עם ה-AI מצורפת לפוסט ותפורסם יחד עם הכיתוב"*.
- הסרה ידנית מכובדת — לא נכפית חזרה בפתיחה הבאה (`seededProducedRef`).
- פותר גם את אזהרת אינסטגרם ("דורש תמונה") — יש תמונה כברירת מחדל.

### כפתור "דוגמה של הפוסט" + `PostPreviewModal`
- כפתור בצבעי פייסבוק (#1877F2) מתחת לאזור המדיה; disabled כשאין כיתוב ואין מדיה.
- מודאל RTL שמדמה פוסט בפיד של פייסבוק לפי העיצוב האמיתי:
  - רקע פיד #F0F2F5, כרטיס לבן עם צל.
  - Header: לוגו המותג כאווטאר עגול 40px (fallback: עיגול כחול עם האות הראשונה), שם העמוד (#050505, 15px, semibold), מתחת "עכשיו · 🌐" (#65676B, 13px), אייקון ··· בצד.
  - **הכיתוב למעלה** (15px, #050505, whitespace-pre-wrap), **התמונה מתחת** ברוחב מלא בלי padding; 2+ תמונות → גריד 2 עמודות עם overlay ‎+N.
  - שורת מעורבות: עיגול לייק כחול #1877F2 עם אגודל לבן + "אתם ועוד 12", ומנגד "3 תגובות · שיתוף אחד".
  - מפריד #CED0D4 ושורת פעולות: אהבתי / תגובה / שיתוף (#65676B, hover #F2F2F2).
- פרטי המותג (שם + לוגו) נטענים lazy מ-supabase (`brands` + bucket `branding`).

## 4. פריסות ואימותים

- **נפרסו** (פעמיים — אחרי המימוש ואחרי התאמות Codex): `process-request`, `twilio-webhook`, `simulator-message`, `send-output`, `generate-presentation`, `edit-image` (project ref `tgropjisnheppsxejfdn`).
- אין מיגרציות DB — נעשה שימוש חוזר ב-`outputs.text_content` (היה NULL לתמונות; נבדק שאין תופעות לוואי ב-FilesPage / מייל / וואטסאפ).
- **אומת E2E**: הפקת תמונה אמיתית → פוסט עברי מלא נשמר על התוצר; עריכת AI ("לקצר + להוסיף טלפון") עבדה מדויק ונשמרה; שרד רענון; המודאל מתמלא אוטומטית; שמירה ידנית (`save_only`) עובדת. כל נתוני הבדיקה (משתמש זמני, תוצר דמה) נמחקו.
- שינויי המודאל (סעיף 3) — client-only, לפי בקשת המשתמש לא הורצו עליהם בדיקות (tsc + vite build נקיים).

## נקודות לתשומת לב

- כל תמונה חדשה כוללת עכשיו קריאת GPT נוספת (הפוסט) — עלות טקסט קטנה שנרשמת ב-usage.
- בוואטסאפ מגיעה הודעת "טקסט מוכן לפוסט" אחרי כל תמונה — אם רוצים לבטל/לשנות ניסוח, זה ב-`deliverContentToWhatsApp` ב-worker.ts.
- תצוגת הפייסבוק היא המחשה סטטית (מספרי לייקים/תגובות דמה).
