# כפתור איפוס שיחה חיה ב־WhatsApp

תאריך: 10.07.2026

## מטרת השינוי

לתת למנהל אפשרות לאפס שיחת WhatsApp חיה ישירות ממסך ההגדרות, בלי צורך לגעת ידנית
בבסיס הנתונים. עד היום איפוס שיחה תקועה חייב הרצת שאילתה ידנית על טבלת `conversations`.

מעכשיו יש כפתור **"אפס שיחה חיה"** בטאב **WhatsApp** במסך *הגדרות מערכת*.

## מה הכפתור עושה

מבצע בדיוק את אותו איפוס שה־flow עושה כשהמשתמש כותב "התחל מחדש"
(ראו `resetConversation` ב־`supabase/functions/_shared/inbound.ts`):

- סוגר בקשה פתוחה (`requests.status = 'closed'`) — אך **לא** בקשה שכבר במצב
  `sent` / `approved` / `sending`, כדי לא לפגוע בתוצר שכבר נשלח.
- מנקה את מצב השיחה: `current_request_id = null`, `flow_state = null`,
  `flow_context = {}`, `selected_output_type = null`, `status = 'active'`.
- ההודעה הבאה של הלקוח פותחת מחדש את התפריט הראשי.

הפעולה חלה על **כל השיחות החיות האמיתיות** ב־WhatsApp
(`simulated = false`, `whatsapp_from LIKE 'whatsapp:%'`, סטטוס
`active` / `waiting_for_user` / `soft_closed`). שיחות סימולטור / e2e לא מושפעות.

## רכיבים שנוספו

### Edge Function: `reset-conversation`

`supabase/functions/reset-conversation/index.ts`

- מאמת שהקורא הוא מנהל (זהה לתבנית של `delete-user`): מזהה מה־JWT ומאמת
  `profiles.role = 'admin'`. קורא שאינו מנהל מקבל `forbidden` / `unauthorized`.
- מקבל גוף אופציונלי `{ phone }` — אם נשלח, מאפס רק את המספר הזה
  (לדוגמה `"whatsapp:+972500000000"`); אחרת מאפס את כל השיחות החיות.
- רושם `logEvent` עם `action: 'conversation_reset'`, `reason: 'admin_button'`
  ומספר השיחות שאופסו.
- מחזיר `{ ok: true, reset: <count>, phones: [...] }`.

### מסך ההגדרות

`src/pages/admin/SettingsPage.tsx`

- כפתור בטאב WhatsApp עם דיאלוג אישור (`window.confirm`) לפני האיפוס.
- קורא ל־`functions.invoke('reset-conversation', { body: {} })` ומציג הודעת
  משוב ("אופסו N שיחות" / "אין כרגע שיחות פעילות לאיפוס" / הודעת שגיאה).

## בדיקות שבוצעו

- `npx tsc --noEmit` — עובר נקי.
- הפונקציה נפרסה לפרודקשן (`supabase functions deploy reset-conversation`).
- בדיקת אבטחה מול הפונקציה החיה: קריאה ללא טוקן וקריאה עם anon-JWT מחזירות
  `unauthorized` — האימות אוכף כמצופה.
- לוגיקת האיפוס עצמה אומתה מראש מול טבלת `conversations` (אותו סט שדות).

## פריסה

```bash
supabase functions deploy reset-conversation
```

הפרונטאנד נפרס כרגיל דרך צינור ה־build הקיים.
