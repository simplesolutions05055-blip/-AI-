# כפתורים אינטראקטיביים ב־WhatsApp

תאריך: 10.07.2026

## מטרת השינוי

להוסיף לבוט ה־WhatsApp של PrimeOS אפשרויות לחיצות, בלי למחוק או לשבור את ה־flow הטקסטואלי הקיים.

המערכת תומכת מעכשיו בשני מצבים הנשלטים ממסך האדמין:

- **המתג כבוי** — השיחה פועלת בדיוק כמו לפני השינוי, עם תפריטים ממוספרים בטקסט.
- **המתג פעיל** — בחירות קצרות מוצגות כ־Quick Reply Buttons ותפריטים ארוכים מוצגים כ־List Picker.

ברירת המחדל והמצב שנשארו בפרודקשן בסיום העבודה הם **כבוי**.

## השינוי במסך האדמין

במסך **הגדרות מערכת → WhatsApp** נוסף מתג בשם:

> כפתורים אינטראקטיביים

הערך נשמר בטבלת `settings` תחת המפתח:

```json
{
  "key": "whatsapp_interactive_messages",
  "value_json": {
    "enabled": false
  }
}
```

השינוי נכנס לתוקף לאחר לחיצה על **שמירת הגדרות** ואינו דורש deploy נוסף.

ה־Content SIDs שנוצרים בזמן ריצה נשמרים תחת מפתח פנימי:

```text
whatsapp_interactive_content_cache
```

המפתח הפנימי אינו נטען לטופס האדמין, כדי ששמירה מאוחרת של ההגדרות לא תדרוס cache חדש שנוצר על ידי Edge Function.

## סוגי ההודעות

### List Picker

משמש כאשר יש יותר משלוש אפשרויות:

- התפריט הראשי: תמונה/פוסט, מסמך, מצגת, תזמון פוסט.
- תפריט הפעולות אחרי קבלת תוצר.

Twilio ו־WhatsApp תומכים בעד 10 פריטים ברשימה בתוך חלון שירות פעיל של 24 שעות.

### Quick Reply

משמש כאשר יש עד שלוש אפשרויות:

- מקור התוכן לתזמון: כתיבה בצ׳אט או תוצר קיים באתר.
- רשת לפרסום: פייסבוק, אינסטגרם או שתיהן.

הכותרת שהמשתמש לוחץ עליה חוזרת ל־webhook כ־`ButtonText`. ה־webhook תומך גם ב־`Body` וב־`ButtonPayload` כגיבוי.

### טקסט חופשי

שאלות הדורשות תוכן משתנה נשארו טקסטואליות, לדוגמה:

- בריף לתוצר.
- הוראות תיקון לתמונה או למסמך.
- תאריך ושעה.
- כתובת מייל.
- אישור המכיל תאריך, כתובת או מידע דינמי.

כך נמנעת יצירה בלתי מוגבלת של Content Templates דינמיים בחשבון Twilio.

## אופן השליחה ל־Twilio

בעת השליחה הראשונה של תפריט אינטראקטיבי מסוים:

1. המערכת מחשבת fingerprint קבוע מתוכן ההודעה והאפשרויות.
2. היא בודקת אם כבר קיים `ContentSid` ב־cache.
3. אם לא קיים, היא יוצרת Content Resource ב־Twilio Content API.
4. ה־`ContentSid` נשמר בטבלת `settings` לשימוש חוזר.
5. ההודעה נשלחת דרך Messages API עם `ContentSid`.

התפריטים הם קבועים, ולכן נוצרת כמות סופית וקטנה של Content Resources.

## fallback ובטיחות

אם יצירת ה־Content Resource או שליחת ההודעה האינטראקטיבית נכשלות:

1. נכתב log בשם `whatsapp_interactive_fallback` ברמת `warning`.
2. המערכת שולחת מיד את הודעת הטקסט הממוספרת הישנה.
3. מצב השיחה מתקדם רק אם אחת מדרכי השליחה הצליחה.

מחוץ לחלון השירות של 24 שעות ההתנהגות הקיימת נשמרה: נשלחת תבנית WhatsApp מאושרת אם הוגדרה, ולא נשלח List Picker שאינו נתמך מחוץ לחלון.

כיבוי המתג אינו משנה את פענוח התשובות. גם כשהמתג פעיל המשתמש עדיין יכול להקליד מספר או מילה במקום ללחוץ.

## התאמת הסימולטור

לטבלת `messages` נוספה העמודה:

```sql
interactive_json jsonb
```

העמודה שומרת את סוג הבקרה והאפשרויות שהוצגו. `simulator-message` מחזיר את המידע ל־React, והסימולטור מציג כפתורים לחיצים מתחת לבועת הבוט.

לחיצה בסימולטור שולחת את כותרת האפשרות דרך אותו endpoint ואותו מנוע שיחה של WhatsApp. כאשר המתג כבוי, `interactive_json` נשאר `NULL` והסימולטור מציג את הודעת הטקסט הישנה.

## קבצים ששונו

- `src/pages/admin/SettingsPage.tsx` — מתג האדמין והגנה על ה־cache הפנימי.
- `src/pages/admin/SimulatorPage.tsx` — הצגת אפשרויות לחיצות בסימולטור.
- `supabase/functions/_shared/twilio.ts` — יצירת Content Resources ושליחה לפי `ContentSid`.
- `supabase/functions/_shared/worker.ts` — קריאת המתג, cache, fallback ושמירת metadata.
- `supabase/functions/_shared/flow.ts` — הגדרת התפריטים האינטראקטיביים וחיבורם ל־state machine.
- `supabase/functions/_shared/inbound.ts` — העברת metadata אינטראקטיבי לשכבת השליחה.
- `supabase/functions/twilio-webhook/index.ts` — קבלת `ButtonText` ו־`ButtonPayload`.
- `supabase/functions/simulator-message/index.ts` — החזרת `interactive_json` ל־frontend.
- `supabase/migrations/20260710123935_whatsapp_interactive_messages.sql` — הוספת העמודה למסד הנתונים.
- `src/components/GptImagesDeck.tsx` — שני type casts חסרי השפעה התנהגותית, כדי לתקן שגיאות TypeScript קיימות ולאפשר quality gate מלא.

### בדיקת E2E מול Supabase המרוחק

הבדיקה הפעילה זמנית את המתג, פתחה שיחת סימולטור חדשה ושלחה `היי`:

```text
interactive_kind=list_picker
interactive_option_count=4
first_choice=תמונה / פוסט
choice_advanced_to_brief=true
```

לאחר מכן המתג כובה ונפתחה שיחה חדשה:

```text
plain_interactive=missing
plain_numbered_menu=true
```

כלומר:

- במצב פעיל התקבלה רשימה אינטראקטיבית עם ארבע אפשרויות.
- בחירה באפשרות הראשונה קידמה את ה־state machine לשלב הבריף.
- במצב כבוי לא הוחזר metadata אינטראקטיבי והתקבל התפריט הממוספר הישן.
- המתג הוחזר ל־`false` בסיום הבדיקה.
